개인정보 처리방침

시행일: 2026-05-04 · 최종 수정: 2026-06-04

Arasaco(이하 "회사")는 이용자의 개인정보를 중요시하며, 「개인정보 보호법」 등 관련 법령을 준수합니다. 본 처리방침은 회사가 운영하는 BuddyRun 모바일 애플리케이션(이하 "앱")이 어떤 정보를 수집하고, 어떻게 사용·보관·보호하는지 설명합니다.

1. 수집하는 정보

1.1 자동으로 수집되는 정보

  • 잠금 세션 데이터: 잠금 시작/종료 시각, 잠금 지속 시간, 도전 성공/실패 여부, 가상 마라톤 거리, 사용자가 입력한 메모 — 기기 내부에만 저장. 가상 마라톤 거리는 잠금 경과 시간에 일정 속도를 곱해 산출하는 시각화용 수치이며, GPS·모션 센서·실제 위치 정보는 일절 사용하지 않습니다.
  • 잠금 스케줄: 사용자가 등록한 반복 잠금 시간표 (이름, 요일, 시작/종료 시각, 활성 여부) — 기기 내부에만 저장
  • 설치된 앱 정보: 사용자가 "허용 앱"을 선택할 수 있도록 Android 런처 앱 목록에서 앱 이름·패키지명을 표시. 외부 전송 없음
  • 앱 사용 통계: 잠금 도전 중 차단할 앱 감지를 위해 Android Usage Stats API 사용. 외부 전송 없음
  • 닉네임: 온보딩에서 사용자가 입력한 앱 내 표시 이름 — 기기 내부에만 저장
  • 크래시 진단 정보: 출시 빌드에서 앱이 비정상 종료될 경우 Firebase Crashlytics를 통해 크래시 로그, 기기/OS 정보, 앱 버전, 오류 시각 등 진단 정보가 전송될 수 있습니다.

1.2 수집하지 않는 정보

  • 이름, 이메일, 전화번호 등 개인 식별 정보
  • 위치 정보 (GPS, IP 위치)
  • 연락처, 통화기록, 메시지
  • 마이크, 카메라, 사진
  • 금융 정보

2. 정보 이용 목적

  • 잠금 도전 기능 제공 (도전 시작·유지·종료)
  • 통계 페이지에서 사용자에게 과거 잠금 기록 표시
  • 스케줄된 잠금이 지정된 시각에 시작되도록
  • 비정상 종료 원인을 파악하고 앱 안정성을 개선

3. 정보 보관 위치 및 기간

잠금 기록, 스케줄, 허용 앱 목록, 설정, 닉네임은 이용자의 기기 내부에 저장됩니다. 단, 출시 빌드에서 앱이 비정상 종료되는 경우 크래시 진단 정보가 Firebase Crashlytics로 전송될 수 있습니다.

4. 제3자 제공 및 위탁

회사는 이용자의 개인정보를 외부에 판매하지 않습니다. 앱 안정성 개선을 위해 Google Firebase Crashlytics를 크래시 진단 처리자로 사용할 수 있으며, 광고 SDK는 사용하지 않습니다.

향후 클라우드 백업 등 새로운 제3자 처리자가 도입될 경우, 본 처리방침을 사전에 갱신하고 시행일 30일 전 앱 내 공지 또는 본 페이지에서 안내합니다.

5. 권한 사용 안내

  • 다른 앱 위에 그리기 (SYSTEM_ALERT_WINDOW): 잠금 도전 중 잠금 화면 표시
  • 사용 데이터 액세스 (PACKAGE_USAGE_STATS): 잠금 중 차단 대상 앱 감지
  • 정확한 알람 (SCHEDULE_EXACT_ALARM): 사용자가 설정한 시각에 정확하게 잠금 시작
  • 포그라운드 서비스 (FOREGROUND_SERVICE_SPECIAL_USE): 잠금 도전 종료 시각까지 잠금 상태 유지
  • 알림 (POST_NOTIFICATIONS): 잠금 진행 상황 알림 표시
  • 부팅 후 실행 (RECEIVE_BOOT_COMPLETED): 재부팅 후 예약된 잠금 복원

6. 이용자 권리
